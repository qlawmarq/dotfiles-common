# パフォーマンスレビューガイド

## このガイドの対象

コードのパフォーマンス特性を評価する際に使用する。
アルゴリズム効率、メモリ使用量、I/O 最適化、データベースクエリ、
フロントエンド描画パフォーマンスなどが対象。

## チェックリスト

### アルゴリズム・計算量

- [ ] 時間計算量が要件に対して適切か（O(n^2) 以上の処理にデータ量の上限は考慮されているか）
- [ ] 空間計算量が適切か（大量データをメモリに展開していないか）
- [ ] 不要なループのネストがないか
- [ ] 同じ計算が繰り返し実行されていないか（メモ化の検討）
- [ ] ソート済みデータに対して線形探索をしていないか（二分探索の検討）

### データベース・クエリ

- [ ] N+1 クエリ問題が発生していないか
- [ ] 適切なインデックスが存在するか（WHERE 句、JOIN 条件、ORDER BY 句のカラム）
- [ ] SELECT * ではなく必要なカラムのみ取得しているか
- [ ] 大量データの取得にページネーションが実装されているか
- [ ] トランザクションの範囲が最小限か（長時間のロック回避）
- [ ] 不要なクエリが実行されていないか（キャッシュの検討）

### メモリ管理

- [ ] 大きなオブジェクトやコレクションが適切に解放されているか
- [ ] メモリリークの可能性がないか（イベントリスナーの未解除、クロージャ内の参照保持）
- [ ] ストリーム処理が可能な場面で全データをメモリに読み込んでいないか
- [ ] バッファサイズが適切に設定されているか

### I/O・ネットワーク

- [ ] 同期 I/O がパフォーマンスボトルネックになっていないか
- [ ] API コール数が最小限か（バッチリクエストの検討）
- [ ] 外部サービス呼び出しにタイムアウトが設定されているか
- [ ] 並列実行可能な I/O が直列化されていないか（Promise.all 等の活用）
- [ ] 適切なキャッシュ戦略が実装されているか（TTL、キャッシュ無効化）

### フロントエンド（該当する場合）

- [ ] 不要な再レンダリングが発生していないか
- [ ] 大きなリストに仮想スクロールが検討されているか
- [ ] 画像やアセットが適切に最適化されているか
- [ ] バンドルサイズが適切か（不要なライブラリの読み込みがないか）
- [ ] レイアウトスラッシング（強制同期レイアウト）が発生していないか

## レビュー手順

1. **データ量の見積もり**: 処理対象のデータ量（現在値と将来の増加見込み）を確認する
2. **ホットパスの特定**: 頻繁に実行されるコードパスを特定する
3. **計算量の分析**: ホットパスのアルゴリズム計算量を分析する
4. **I/O パターンの確認**: データベースクエリや外部 API 呼び出しのパターンを確認する
5. **公式ドキュメントの参照**: フレームワーク固有の最適化ガイドを参照する
6. **ベンチマーク提案**: パフォーマンスが懸念される箇所にはベンチマークの実施を提案する

## よくある問題パターン

### N+1 クエリ問題

- **症状**: ループ内でデータベースクエリを実行（例: 各ユーザーの投稿をループで取得）
- **根拠**: N+1 は最も頻繁に発生するパフォーマンス問題の一つ
- **対処**: Eager Loading (JOIN / INCLUDE) またはバッチクエリに変更する。例:

```python
# N+1 問題
for user in users:
    posts = db.query("SELECT * FROM posts WHERE user_id = ?", user.id)

# 改善: JOIN で一括取得
users_with_posts = db.query("""
    SELECT u.*, p.* FROM users u
    LEFT JOIN posts p ON u.id = p.user_id
""")
```

### 同期的な直列処理

- **症状**: 独立した複数の I/O 操作を順番に実行
- **根拠**: 独立した I/O は並列実行で大幅に高速化できる
- **対処**: `Promise.all`, `asyncio.gather`, `goroutine` 等で並列化する。例:

```javascript
// 直列（遅い）
const users = await fetchUsers();
const products = await fetchProducts();
const orders = await fetchOrders();

// 並列（速い）
const [users, products, orders] = await Promise.all([
    fetchUsers(),
    fetchProducts(),
    fetchOrders(),
]);
```

### メモリリーク（イベントリスナー）

- **症状**: コンポーネントの破棄時にイベントリスナーが解除されていない
- **根拠**: 解除されないリスナーは GC を阻害し、メモリリークの原因となる
- **対処**: クリーンアップ処理（`removeEventListener`, `useEffect` の return, `AbortController`）を追加する。

### 全件メモリ読み込み

- **症状**: CSV ファイルや大きなクエリ結果を全件メモリに読み込む
- **根拠**: データ量増加時に OOM (Out of Memory) の原因となる
- **対処**: ストリーム処理、カーソルベースのイテレーション、バッチ処理に変更する。
