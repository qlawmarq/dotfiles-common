# セキュリティレビューガイド

## このガイドの対象

コードのセキュリティ上の脆弱性やリスクを評価する際に使用する。
Web アプリケーション、API、バックエンド処理、インフラ設定など、
セキュリティに影響するすべてのコードが対象。

## チェックリスト

### 入力バリデーション

- [ ] ユーザー入力が適切にバリデーションされているか
- [ ] SQL インジェクション対策: パラメータ化クエリ / プリペアドステートメントを使用しているか
- [ ] XSS 対策: 出力時のエスケープ / サニタイズが行われているか
- [ ] コマンドインジェクション対策: シェルコマンドの構築にユーザー入力を直接使用していないか
- [ ] パストラバーサル対策: ファイルパスにユーザー入力を使用する際にバリデーションしているか
- [ ] SSRF 対策: URL にユーザー入力を使用する際にホワイトリスト検証しているか
- [ ] デシリアライゼーション: 信頼できないデータの安全なデシリアライゼーションか

### 認証・認可

- [ ] 認証が必要なエンドポイントに認証チェックがあるか
- [ ] 認可チェックが適切に実装されているか（RBAC/ABAC）
- [ ] 水平権限昇格: 他ユーザーのリソースへのアクセスが防止されているか
- [ ] 垂直権限昇格: 権限の低いユーザーが高権限の操作を行えないか
- [ ] セッション管理が適切か（有効期限、再生成、無効化）
- [ ] JWT を使用している場合: 署名検証、有効期限チェック、アルゴリズム指定が適切か

### シークレット管理

- [ ] ハードコードされたシークレット（パスワード、API キー、トークン）がないか
- [ ] シークレットがログに出力されていないか
- [ ] シークレットがエラーメッセージに含まれていないか
- [ ] `.env` ファイルや設定ファイルがバージョン管理から除外されているか
- [ ] シークレットが環境変数またはシークレットマネージャー経由で取得されているか

### 暗号化・データ保護

- [ ] 通信が TLS/HTTPS で保護されているか
- [ ] パスワードが適切にハッシュ化されているか（bcrypt, Argon2 等。MD5/SHA-1 は不可）
- [ ] 暗号化アルゴリズムが最新の推奨に沿っているか（AES-256-GCM 等）
- [ ] 暗号鍵が適切に管理されているか
- [ ] 個人情報（PII）が適切に保護されているか

### エラーハンドリングと情報漏洩

- [ ] エラーメッセージにスタックトレースやシステム内部情報が含まれていないか
- [ ] デバッグモードが本番環境で無効化されているか
- [ ] エラーレスポンスが攻撃者に有用な情報を提供していないか
- [ ] ログに機密情報が出力されていないか

### 依存関係のセキュリティ

- [ ] 既知の脆弱性がある依存パッケージを使用していないか
- [ ] 依存関係のバージョンが固定されているか（lock ファイル）
- [ ] 使用していない依存パッケージが含まれていないか

### API セキュリティ

- [ ] レート制限が実装されているか
- [ ] CORS 設定が適切か（ワイルドカード `*` の不適切な使用がないか）
- [ ] HTTP ヘッダーのセキュリティ設定が適切か（CSP, X-Frame-Options 等）
- [ ] リクエストサイズの制限があるか
- [ ] 機密データが URL パラメータに含まれていないか（POST ボディを使用すべき）

## レビュー手順

1. **信頼境界の特定**: システムのどこで信頼できないデータが入ってくるかを特定する
2. **データフローの追跡**: 外部入力がシステム内でどのように流れ、処理されるかを追跡する
3. **各チェックポイントの確認**: データが信頼境界を越える箇所でチェックリストを適用する
4. **公式セキュリティガイドの参照**: 使用しているフレームワークのセキュリティガイドを参照し、推奨パターンに沿っているか確認する
5. **依存関係の確認**: `npm audit`, `pip-audit`, `cargo audit` 等のツールの結果を確認する
6. **脆弱性データベースの照合**: 使用パターンが既知の脆弱性パターン（CWE）に該当しないか確認する

## よくある問題パターン

### SQL インジェクション (CWE-89)

- **症状**: 文字列連結で SQL クエリを構築
- **根拠**: OWASP Top 10 — A03:2021 Injection
- **対処**: パラメータ化クエリまたは ORM を使用する。例:

```python
# 危険
cursor.execute(f"SELECT * FROM users WHERE id = {user_id}")

# 安全
cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))
```

### XSS (CWE-79)

- **症状**: ユーザー入力をそのまま HTML に挿入
- **根拠**: OWASP Top 10 — A03:2021 Injection
- **対処**: テンプレートエンジンの自動エスケープ機能を使用する。`dangerouslySetInnerHTML` や `v-html` の使用は必ず検証する。

### 安全でないデシリアライゼーション (CWE-502)

- **症状**: 信頼できないソースからのデータを `eval()`, `pickle.loads()`, `unserialize()` 等で処理
- **根拠**: OWASP Top 10 — A08:2021 Software and Data Integrity Failures
- **対処**: JSON 等の安全なフォーマットを使用し、スキーマバリデーションを行う。

### ハードコードされたシークレット (CWE-798)

- **症状**: ソースコード内に API キー、パスワード、トークンが直接記述
- **根拠**: OWASP Top 10 — A07:2021 Identification and Authentication Failures
- **対処**: 環境変数またはシークレットマネージャーに移行する。

### 不適切なアクセス制御 (CWE-284)

- **症状**: リソースへのアクセス時に所有者チェックが欠落
- **根拠**: OWASP Top 10 — A01:2021 Broken Access Control
- **対処**: 全てのリソースアクセスで認可チェックを実装する。
