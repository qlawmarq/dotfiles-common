# アクセシビリティレビューガイド

## このガイドの対象

ユーザーインターフェース（Web、モバイル、デスクトップアプリ）のアクセシビリティを評価する際に使用する。
HTML/CSS、React/Vue 等のフロントエンドコンポーネント、フォーム、ナビゲーションなどが対象。

**注意**: バックエンドのみの変更にはこのガイドは適用しない。

## チェックリスト

### セマンティック HTML

- [ ] 見出し階層が論理的か（h1 → h2 → h3、レベルの飛ばしがないか）
- [ ] ランドマーク要素が適切に使用されているか（`<nav>`, `<main>`, `<aside>`, `<footer>`）
- [ ] リストが `<ul>`, `<ol>`, `<dl>` で適切にマークアップされているか
- [ ] `<div>` や `<span>` がインタラクティブ要素として誤用されていないか（`<button>`, `<a>` を使用すべき）
- [ ] テーブルに適切なヘッダー（`<th>`, `scope`）があるか

### キーボード操作

- [ ] 全てのインタラクティブ要素にキーボードでアクセスできるか
- [ ] フォーカス順序が論理的か（`tabindex` の不適切な使用がないか）
- [ ] フォーカスインジケーターが視覚的に確認できるか（`outline: none` の不用意な使用がないか）
- [ ] キーボードトラップが存在しないか（モーダル内のフォーカス管理は例外）
- [ ] カスタムウィジェットに適切なキーボードイベントハンドラがあるか

### ARIA (Accessible Rich Internet Applications)

- [ ] ネイティブ HTML 要素で十分な場合に不要な ARIA が使用されていないか（"No ARIA is better than bad ARIA"）
- [ ] ARIA ロールが正しく使用されているか
- [ ] `aria-label` / `aria-labelledby` で適切なラベルが提供されているか
- [ ] 動的コンテンツの更新が `aria-live` で通知されるか
- [ ] `aria-hidden="true"` が視覚的に表示されているコンテンツに誤って適用されていないか

### フォームアクセシビリティ

- [ ] 全てのフォーム入力に関連付けられた `<label>` があるか
- [ ] 必須フィールドが `aria-required` またはテキストで明示されているか
- [ ] エラーメッセージが入力フィールドに `aria-describedby` で関連付けられているか
- [ ] エラー発生時にフォーカスがエラー箇所に移動するか
- [ ] プレースホルダーがラベルの代わりに使用されていないか

### 視覚的アクセシビリティ

- [ ] テキストと背景のコントラスト比が WCAG AA 基準を満たしているか（通常テキスト 4.5:1、大きなテキスト 3:1）
- [ ] 色だけで情報を伝えていないか（色覚多様性への配慮）
- [ ] テキストサイズが 200% に拡大しても情報が欠落しないか
- [ ] 動くコンテンツに一時停止/停止のコントロールがあるか
- [ ] 3回/秒以上の点滅がないか

### 画像・メディア

- [ ] 意味のある画像に適切な `alt` テキストがあるか
- [ ] 装飾的な画像に `alt=""` または `aria-hidden="true"` が設定されているか
- [ ] 動画にキャプション（字幕）が提供されているか
- [ ] 音声コンテンツにトランスクリプト（文字起こし）があるか

## レビュー手順

1. **UI コンポーネントの特定**: 変更に含まれる UI コンポーネントを特定する
2. **セマンティクスの確認**: HTML 構造が意味的に正しいか確認する
3. **キーボード操作の検証**: タブ順序やフォーカス管理を検証する
4. **ARIA 使用の検証**: ARIA 属性の使用が適切か確認する。[WAI-ARIA Authoring Practices](https://www.w3.org/WAI/ARIA/apg/) の該当パターンを参照する
5. **コントラストの確認**: 色のコントラスト比が基準を満たすか確認する
6. **フレームワーク固有のガイド参照**: React, Vue 等のアクセシビリティ公式ガイドを参照する

## よくある問題パターン

### クリック可能な div

- **症状**: `<div onClick={...}>` でボタンやリンクを実装
- **根拠**: WCAG 2.1 Success Criterion 4.1.2 — インタラクティブ要素はネイティブ要素を使用すべき
- **対処**: `<button>` または `<a>` に変更する。やむを得ず `<div>` を使用する場合は `role="button"`, `tabindex="0"`, キーボードイベントを追加する。

### フォーカスインジケーターの削除

- **症状**: `outline: none` が `:focus-visible` の代替なく適用されている
- **根拠**: WCAG 2.1 Success Criterion 2.4.7 — フォーカスインジケーターは必須
- **対処**: `:focus-visible` でカスタムフォーカススタイルを提供する。

### 画像の alt 属性欠落

- **症状**: `<img>` に `alt` 属性がない
- **根拠**: WCAG 2.1 Success Criterion 1.1.1 — 非テキストコンテンツには代替テキストが必要
- **対処**: 内容を説明する `alt` テキストを追加する。装飾画像には `alt=""` を設定する。

### aria-label の濫用

- **症状**: ネイティブ HTML で十分なラベル付けが可能な箇所に `aria-label` を使用
- **根拠**: W3C — "No ARIA is better than bad ARIA"。ネイティブ HTML のセマンティクスが最優先
- **対処**: `<label>`, `<caption>`, `<legend>` 等のネイティブ HTML 要素でラベル付けする。
