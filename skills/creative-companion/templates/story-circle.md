# Dan Harmon Story Circle

> **モード: 構造化** — Joseph Campbell の Hero's Journey 17 段階を 8 段階に圧縮した版。Harmon（『Community』『Rick and Morty』）が自身の脚本作成プロセスとして公開。30 分で骨格ができる。

縦軸 = 内的変化、横軸 = 外的変化。

## 8 段階

| # | 段階 | 質問 | 記入欄 |
|---|------|------|--------|
| 1 | **You** | キャラは安定領域にいる | |
| 2 | **Need** | 何かが欠けている／必要としている | |
| 3 | **Go** | 未知の領域に踏み出す | |
| 4 | **Search** | 適応・苦闘する | |
| 5 | **Find** | 求めていたものを得る | |
| 6 | **Take** | しかし対価を支払う | |
| 7 | **Return** | 元の世界に戻る | |
| 8 | **Change** | 変化した自分として戻る | |

各段階を 1-2 文で埋める。完成すると 8-16 文の物語骨格。

## 一貫性チェック

- [ ] 1 と 8 を並べたとき、主人公の**内的変化**が明示されているか？
- [ ] 3 と 7 を並べたとき、主人公の**外的世界の変化**が分かるか？
- [ ] 5 と 6 のセット（得たもの／その対価）が**1 対 1 で対応**しているか？
- [ ] 2 の「欠け」が 8 の「変化」で満たされているか？満たされないなら、なぜ満たされないかが物語の核

## Snowflake との関係

Snowflake ステップ 2 の 1 段落（5 文）と並行で書くと整合が取りやすい:

- Snowflake「状況」≒ Circle 1-2
- Snowflake「危機 1」≒ Circle 3-4
- Snowflake「危機 2」≒ Circle 4-5
- Snowflake「危機 3」≒ Circle 6
- Snowflake「結末」≒ Circle 7-8

矛盾があれば Circle の方を優先する（より具体的だから）。
